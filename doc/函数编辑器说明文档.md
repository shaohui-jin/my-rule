# 函数编辑器说明文档

## 概述

「函数编辑」页面用于编写带 **JSDoc 风格注释** 的 **JavaScript** 函数，解析注释后生成表单配置，实现注释与表单的双向驱动。路由为 `/func`（`src/router/modules/rule.ts`），页面组件为 `src/views/func.vue`，菜单标题为「函数JSDoc解析器」。

> 说明：旧版文档中描述的 Lua `-- @param` 注释、`luaparse` AST、组件名 `FuncEdit.vue` / `LuaEditor` 等**与当前仓库不一致**，已废弃。

## 页面与依赖

| 职责 | 实现 |
|------|------|
| 代码编辑 | `BaseEditor`（`src/components/base/BaseEditor.vue`），基于 Monaco |
| 注解解析结果展示 | `BasePre`（`src/components/base/BasePre.vue`） |
| 入参/出参表单预览 | `BaseFormRender`（`src/components/base/BaseFormRender.vue`） |
| 注释 → 结构化数据 | `JSDocParser`（`src/utils/parser/JSDocParser.ts`） |
| 结构化数据 → 表单配置 | `FormParser`（`src/utils/parser/FormParser.ts`） |

内置模版内容来自 `getTemplate()`（同文件内），为 **JSDoc** 示例（`@param`、`@returns`、`@example` 等），而非 Lua。

## Monaco 默认配置

公共默认配置位于 **`src/utils/config/MonacoEditor.ts`** 的 `getDefaultMonacoEditorConfig`：

- 语言：`javascript`（文件中保留了对 `lua` 的注释说明）  
- 主题：`vs`  
- **行号关闭**（`lineNumbers: 'off'`），与画布侧表达式编辑体验一致  
- 自动换行、折叠、快速建议等  

## 工作流中的表单渲染

工作流节点属性面板 **`FuncPanel.vue`** 中仍使用 **`SimpleFormRenderer`**（`src/components/funcForm/SimpleFormRenderer.vue`）渲染函数入参，与函数编辑主页的 `BaseFormRender` 是两条使用场景：前者嵌入规则设计器，后者用于 `/func` 页表单预览。

## 注释与解析（JSDoc）

解析逻辑以 **`JSDocParser`**、**`FormParser`** 为准，支持标签包括（页面「工具说明」中已提示）：`@param`、`@return` / `@returns`、`@example` 等。参数行可携带 `#` 后的扩展配置（如 `placeholder`、`value`、`options`、`compType` 等），具体字段与 **`FormParser.ts`** 中 `Config`、`FormAttribute` 及解析实现一致。

若需扩展新标签或新控件类型，请修改：

1. `JSDocParser.ts`：注释扫描与结构化  
2. `FormParser.ts`：`SUPPORTED_COMPONENT_TYPES` 与表单属性映射  

## 典型操作流（`func.vue`）

1. 编辑左侧代码（可点击「模版」载入示例）。  
2. 点击「解析」：`Js2FormJson` 调用 `jsDocParser.parse` 与 `formParser.parse`，更新中间 JSON 与右侧表单。  
3. 「保存」：校验后通过 `http` 提交函数数据（详见页面内 `saveFuncData`）。  

## 表达式输入（画布侧）

条件等场景下的单行/多行表达式编辑使用 **`BaseFunctionInput`**（`src/components/base/BaseFunctionInput.vue`），Monaco 配置同样来自 `getDefaultMonacoEditorConfig`，与本文档前述行号关闭等行为一致。
